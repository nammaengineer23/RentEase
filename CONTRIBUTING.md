# Contributing to RentEase

Thanks for helping improve RentEase.

## Development Workflow

1. Fork the repository and create a feature branch.
2. Keep changes focused on a single concern.
3. Follow existing code style and architecture patterns.
4. Add or update tests where appropriate.
5. Submit a pull request with a clear description.

## Coding Standards

- Prefer readable, maintainable code over clever implementations.
- Keep modules small and responsibilities clear.
- Document public behavior and non-obvious decisions.
- Write tests for bug fixes and new features when feasible.

## Commit Guidelines

- Use clear, descriptive commit messages.
- Keep commits small and logical.
- Reference related issues or tasks when applicable.

## Review Expectations

- Pull requests should include a summary of the change and verification steps.
- Be respectful and constructive during reviews.
- Address feedback promptly and clearly.
